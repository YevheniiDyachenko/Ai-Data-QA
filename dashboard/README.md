# Data Quality Dashboard (Next.js + TypeScript)

A lightweight local dashboard for visualizing JSON output generated by the Python Data QA tool.

## Features

- Table-level test summaries with pass/fail highlighting
- Expandable failed test panels with AI analysis context
- Profiling metrics (row count, null count, distinct count)
- Optional trend chart from historical runs (`dq_history.json`)
- Refresh button to reload report JSON without restarting the app
- Table status filter (`all`, `passing`, `failing`)

## Installation

```bash
cd dashboard
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Link to existing Data QA reports

The dashboard reads from static files in `public/reports/`:

- `public/reports/data_quality_report.json` (required)
- `public/reports/dq_history.json` (optional)

To connect your existing Python QA tool:

1. Configure your Python report generator to write JSON outputs into `dashboard/public/reports/`.
2. Ensure the main report file is named `data_quality_report.json`.
3. (Optional) Add historical trend data as `dq_history.json`.

If your reports are generated in another repository path, copy/sync them into this folder before refreshing the dashboard.

## Usage examples

- Refresh report after rerunning QA checks:
  1. Run Python QA report generation.
  2. Click **Refresh** in the dashboard.

- Focus only on failing tables:
  1. Use the status filter dropdown.
  2. Select **Failing only**.
